Troubleshooting

I can't open the Seismic Line Mapper interface.

SLM relies on Python and ArcGIS to work. Make sure you have the latest version of ArcGIS installed. This installation normally includes a version of Python (2.7 or 3).

Seismic Line Mapper opens but when I run any tool I get an error message.

Make sure you have a valid ArcGIS license as well as the spatial analyst extension. You should be able to import arcpy in any Python console without an error message.

The Canopy Cost Raster tool works but Center Line does not.

Try running the Clip Shape By Raster tool first. The input lines must be contained completely within the input raster without any overlap on NoData cells.

The Center Line tool works but following tools do not.

Check that the input lines' attribute table contains a field for their Least Cost Corridor Threshold (LCCT). The default field is named CorridorTh. When running any tools that ask for a corridor threshold field, check that the name is an exact match. Invalid characters such as spaces may cause problems.

The Line Footprint tool is taking a disproportionately long time to process.

Set Process Segments to False. Increase the number of processing cores.

I get the following error: Cannot get exclusive schema lock.

Close the SLM and all ESRI ArcGIS software. Delete all subfolders in the Scripts folder (but not the scripts) before running any tools. Do not run more than one tool at once.
